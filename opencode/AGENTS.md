# Global OpenCode Rules

These rules apply across all projects and sessions.

## General Behavior

- Be concise. Prefer short, direct responses unless detail is explicitly needed.
- Do not add unsolicited explanations after completing a task — just confirm what was done.
- Ask clarifying questions before making large or irreversible changes.
- Prefer editing existing files over creating new ones.
- Never create documentation files (README, CHANGELOG, etc.) unless explicitly asked.

## Code Style

- Match the conventions already present in the codebase — don't impose external styles.
- Prefer explicit over clever. Readable code beats terse code.
- Remove dead code rather than commenting it out.
- Do not add comments that just restate what the code does.

## Output Format

- Use plain prose for explanations — avoid excessive bullet points.
- When referencing code locations, use the `file:line` format.
- Do not use emojis unless asked.

## Commits

- Do not commit unless explicitly asked.
- Write commit messages that explain *why*, not just *what*.
- Do not use `--no-verify` or skip hooks.
